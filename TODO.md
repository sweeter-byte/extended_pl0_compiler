# PL/0 编译器系统 - 嵌入式与操作系统集成路线图

为了将当前的 PL/0 编译器系统与嵌入式开发及操作系统内核产生联系，拟定以下三个阶段的开发计划：

## 阶段一：串口 I/O 驱动与裸机运行 (Bare-metal)
- [ ] 编写一个简单的“裸机”串口驱动（针对目标嵌入式板子）。
- [ ] 移植 P-Code 解释器，使其不依赖标准库。
- [ ] 让解释器通过串口在 PC 终端上输出 "Hello World"。

## 阶段二：原生汇编代码生成器 (Back-end)
- [ ] 为编译器添加一个简单的汇编生成器（如针对 RISC-V 或 ARM）。
- [ ] 尝试将 PL/0 源码直接编译为可在微控制器上运行的机器码。
- [ ] 在真实硬件或模拟器上验证运行结果。

## 阶段三：操作系统内核集成 (OS Shell)
- [ ] 在操作系统内核中集成 PL/0 编译器/解释器作为运行时环境。
- [ ] 扩展指令集以支持基本的系统调用。
- [ ] 编写一个 PL/0 程序，作为操作系统启动后的第一个 “Shell” 或初始化程序。
